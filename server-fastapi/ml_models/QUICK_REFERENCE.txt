QUICK REFERENCE - Custom Job Matching System

Files Created:
==============

1. ml_models/collect_training_data.py
   - Pulls student applications from database
   - Generates positive (applied) and negative (not applied) examples
   - Creates training_data.csv

2. ml_models/feature_engineering.py
   - Extracts numerical features from student-job pairs
   - Features: skills overlap, text similarity, program match, etc.
   - Reusable for training and prediction

3. ml_models/train_model.py
   - Trains 3 models: Logistic Regression, Random Forest, XGBoost
   - Evaluates each on test set
   - Saves best model as best_model_latest.pkl

4. ml_models/predictor.py
   - MatchingModelPredictor class for loading and using trained model
   - predict() - Single prediction
   - predict_batch() - Multiple predictions

5. routes/ml_matching.py
   - FastAPI endpoints for ML predictions
   - POST /api/match/predict - Single match
   - POST /api/match/predict-batch - Multiple matches
   - GET /api/match/student/{id}/recommendations - Top recommendations
   - GET /api/match/health - Check model status

6. ml_models/quick_start.py
   - One-command script to run all training steps

7. ml_models/README.md
   - Complete documentation with examples

// In your service
getMatchScores(studentId: number, internshipIds: number[]) {
  return this.http.post('/api/match/predict-batch', {
    student_id: studentId,
    internship_ids: internshipIds
  });
}

// In your component
async loadInternshipsWithMatches() {
  const internships = await this.getInternships();
  const matches = await this.getMatchScores(
    this.studentId,
    internships.map(i => i.id)
  );
  
  // Merge matches into internships
  this.internships = internships.map(i => ({
    ...i,
    matchScore: matches.find(m => m.internship_id === i.id)?.match_score
  }));
  
  // Sort by match score
  this.internships.sort((a, b) => (b.matchScore || 0) - (a.matchScore || 0));
}

// In template
<div *ngFor="let job of internships">
  <h3>{{ job.title }}</h3>
  <div class="match-bar">
    <div [style.width.%]="job.matchScore * 100"></div>
  </div>
  <span>{{ job.matchScore * 100 | number:'1.0-0' }}% Match</span>
</div>


Match Score Interpretation:
============================

0.70 - 1.00: Strong Match ⭐⭐⭐
0.50 - 0.69: Good Match ⭐⭐
0.30 - 0.49: Weak Match ⭐
0.00 - 0.29: Poor Match


Features Used:
==============

1. skills_jaccard: Jaccard similarity between student and job skills
2. skills_match_count: Number of matching skills
3. skills_required_count: Total skills required by job
4. skills_coverage: % of required skills student has
5. is_job_placement: 1 if job posting, 0 if internship
6. program_match: Student's program mentioned in job title/description
7. major_match: Student's major mentioned in job title/description
8. text_cosine_similarity: TF-IDF cosine similarity of full text

Troubleshooting:
================

1. "Model not found"
   → Run: python ml_models/train_model.py

2. "Not enough training data"
   → Need at least 20-30 applications
   → Use the system more to collect data

3. "XGBoost not available"
   → Optional: pip install xgboost
   → Random Forest works fine too

4. "Import error in ml_matching.py"
   → Check Python path
   → Make sure ml_models/__init__.py exists

5. Low accuracy
   → Need more training data
   → Encourage students to fill out complete profiles
   → Check feature importance and add more features
